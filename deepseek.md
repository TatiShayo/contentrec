## 🟠 Performance Issues

| Severity | File | Line(s) | Issue | Exact Fix |
|----------|------|---------|-------|-----------|
| 🔴 CRITICAL | `data/database.py` | 71-78 | Every request opens and closes a new SQLite connection. No `check_same_thread=False`, no connection pool, no WAL mode. Under concurrent load → file-lock contention, request timeouts. | Add connection pool (e.g., `sqlite3.connect(db_path, check_same_thread=False)` + `Queue` of 3 connections). Enable WAL: `PRAGMA journal_mode=WAL;`. Or switch to `aiosqlite` for async. |
| 🟠 HIGH | `data/database.py` | 49-54 | `impressions` table has NO indexes on `user_id`, `item_id`, or `timestamp`. Every query scans entire table. | Add: `CREATE INDEX IF NOT EXISTS idx_impressions_user ON impressions(user_id);` `CREATE INDEX IF NOT EXISTS idx_impressions_item ON impressions(item_id);` `CREATE INDEX IF NOT EXISTS idx_impressions_ts ON impressions(timestamp);` |
| 🟠 HIGH | `models/causal.py` | 78-147 | `train_model()` fetches ALL impressions without LIMIT or date filter. On large datasets this blocks training for seconds. | Add `LIMIT 10000` or date-based filter. Implement incremental training so only new impressions are processed. |
| 🟡 MEDIUM | `models/engine.py` | — | No `@lru_cache` on `embed_item()` or feature vector construction. Every recommendation call re-computes embeddings. | Add `@lru_cache(maxsize=1024)` on embedding methods. |
